# CLAUDE.md — labelle-engine

Guidance for Claude Code when working in this directory. See `../CLAUDE.md` for toolkit-wide context.

## What this library is

A pure Zig library for 2D game development. **No `main()` of its own** — every shipped game has its entry point generated by `labelle-assembler` (see `flying-platform-labelle/.labelle/<target>/main.zig` for the canonical shape). Argv parsing lives in that generated main, not here.

## Build commands

```bash
zig build         # build the library module
zig build test    # all engine tests
```

## Source map (high-traffic files)

| File | Role |
|------|------|
| `src/root.zig` | Public module surface. Add new re-exports here so downstream code can reach them via `engine.Foo`. |
| `src/game.zig` | `GameConfig` / `Game` — comptime-parameterised game container (renderer, ECS, input, audio, gui, hooks). |
| `src/sprite_animation.zig` + `sprite_animation_tick.zig` | Reference for the "one type + one tick fn, paired" pattern used elsewhere. |
| `src/jsonc_scene_bridge.zig` | Scene loading from `.jsonc`. |
| `src/preview_mode.zig` | Play-in-Editor control channel (see below). |
| `test/*.zig` | Each test file is its own binary, wired explicitly in `build.zig`. Tests use `std.testing` directly — zspec is available toolkit-wide but not required here. |

## Preview Mode (issue #516, PIE umbrella #59)

Phase 0 spike for the labelle-gui Play-in-Editor preview. The engine ships a `Preview` helper that dials *out* to a TCP endpoint the editor binds (loopback by default) and emits newline-delimited JSON control frames:

```text
{"kind":"hello","engine_version":"…","pid":12345,"protocol_version":1}\n
{"kind":"heartbeat","t":847291}\n
{"kind":"bye","reason":"normal"}\n
```

**Why connect-out (not listen-in)**: the editor already spawns the game binary (see `labelle-gui/src/compiler.zig`), so it owns port selection and lifecycle. The engine just needs the host:port to dial. Decision recorded in labelle-gui#60.

### Where the flag lives

`--preview-mode <host:port>` (or `--preview-mode=<host:port>`) is parsed by the **generated `main.zig`**, NOT inside this library. The engine is a library — argv doesn't reach it directly. The library offers:

- `engine.parsePreviewArgs(argv)` — pulls the host:port out of an argv slice, returns `null` when absent.
- `engine.Preview.connect(alloc, host_port)` — dials the editor.
- `engine.Preview.sendHello / .tickHeartbeat / .sendBye / .deinit` — protocol primitives.

The generated main calls those at init, in the frame loop, and on shutdown. Wiring into the assembler template is a Phase 1 follow-up (this spike PR is library-only).

### Phase 0 scope (DO NOT exceed without a new RFC)

- Three message kinds: `hello`, `heartbeat`, `bye`. No telemetry yet.
- Newline framing. Length-prefixed binary is Phase 2 work.
- No reconnect. If the editor goes away the engine errors and the next write fails — the game can decide whether to keep running or exit.
- No breakpoints / pause / step. Those are Phase 3+.

### Tests

`test/preview_mode_test.zig` exercises the full `connect → hello → heartbeats → bye → EOF` cycle against a real loopback `std.net.Server` in-process. End-to-end with an actual subprocess will land editor-side in labelle-gui Phase 1.

## Conventions

1. **Module + test file pair**: `src/foo.zig` + `test/foo_test.zig`, wired in `build.zig`'s `test_files` array. Do NOT leave `test {}` blocks inside `src/*.zig` (build.zig doesn't reach them through the cross-module import — see the comment above `assets_tests` in `build.zig`).
2. **Re-export through `root.zig`**: every new public type/fn surfaces as `engine.Foo`. Group with a `// ── Section ──` heading.
3. **`std.json.Stringify.valueAlloc`** for any JSON emission — escape correctness matters once the wire is read by another process.
